# Codex Partnership

Haijun should become a close developer partner for Codex inside the JGS
ecosystem.

## Codex Strengths

- active coding in the current workspace;
- reading and editing files;
- running local commands;
- debugging immediate problems;
- creating scripts, docs, and structure.

## Haijun Strengths To Build

- long-term JGS memory;
- owner and ecosystem identity;
- persistent task state;
- security classification;
- training/evaluation history;
- project-specific developer habits.

## Partnership Pattern

Haijun should provide context and continuity. Codex should perform active
workspace work. Together they should:

- avoid repeating old mistakes;
- keep project rules consistent;
- identify next useful work without waiting for detailed instructions;
- preserve owner intent;
- separate safe automation from approval-required actions.

## Clone-Like Compatibility

Haijun should not literally clone Codex. It should become compatible with Codex:

- same JGS terminology;
- shared task memory;
- compatible runbooks;
- compatible patch and review style;
- clear handoff summaries;
- model-agnostic interface so stronger base models can be swapped in.
