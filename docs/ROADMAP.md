# JGS Roadmap

## Phase 0: Foundation

- Establish GitHub structure.
- Connect `juglow.my.id` to Cloudflare DNS.
- Keep Haijun server package deployable.
- Prepare low-cost GPU first-run flow.

## Phase 1: Haicode and Converhai

- Define core Haicode doctrine.
- Map C/C++ and Unix/Linux concepts into `haic*`.
- Build conversion examples.
- Create validation rules for behavior preservation.

## Phase 2: Haijun

- Run server first-run inspection.
- Evaluate base models.
- Curate JGS/Haicode examples.
- Start small QLoRA experiments only after owner approval.

## Phase 3: JUROX OS

- Define OS architecture layers.
- Map syscall, ABI, runtime, filesystem, and shell concepts.
- Plan portable and installable JUROX forms.
